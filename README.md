# 🫀 Heart Disease Risk Prediction — End-to-End ML Project

An interactive machine learning web app that predicts a **calibrated probability** of heart disease
from clinical patient data, and explains **why** for every individual prediction.

**🔗 Live demo:** _add your Streamlit Cloud URL here_
**📦 Dataset:** 918 real, deduplicated patients combined from Cleveland Clinic, Hungarian Institute of
Cardiology, University Hospital Zurich, University Hospital Basel, and the VA Medical Center Long Beach.

---

## What makes this project different from a typical portfolio ML app

Most beginner heart-disease projects train one or two models on the 303-row Cleveland dataset and
stop there. This project instead:

- Uses a **~3x larger combined dataset** (918 clean patients from 5 studies), and actually fixes its
  known data-quality issues (272 exact duplicates removed, 172 zero-encoded missing cholesterol
  values imputed rather than treated as real zeros).
- Tunes **6 algorithms** with `RandomizedSearchCV` (Logistic Regression, Random Forest, Gradient
  Boosting, XGBoost, SVM, Decision Tree) plus a **stacking ensemble**, all under 5-fold stratified
  cross-validation.
- **Calibrates the winning model's probabilities** (`CalibratedClassifierCV`) so the "72% risk" you
  see on screen is a genuinely calibrated estimate, not just a confident-looking raw score — verified
  with a reliability curve and Brier score.
- Explains **every single prediction** with SHAP: the app shows exactly which of *this patient's*
  values pushed their risk up or down, not just generic global feature importance.
- Ships as a fully interactive Streamlit dashboard: live risk gauge, per-patient factor breakdown,
  population comparison, and an explorable scatter plot of the training data.

## Results

| Model | Test ROC-AUC | Test Accuracy | Test F1 |
|---|---|---|---|
| **Gradient Boosting (deployed)** | **0.949** | **91.8%** | **0.927** |
| Stacking Ensemble | 0.948 | 91.3% | 0.922 |
| XGBoost | 0.948 | 90.8% | 0.916 |
| Random Forest | 0.940 | 91.3% | 0.922 |
| SVM | 0.914 | 85.9% | 0.877 |
| Logistic Regression | 0.908 | 85.9% | 0.877 |
| Decision Tree | 0.889 | 82.1% | 0.831 |

Full leaderboard, ROC curve, confusion matrix, and calibration curve are viewable live in the app's
**Model Performance** tab.

## Project structure

```
├── app.py                    # Streamlit app (3 pages: Predict, Model Performance, About)
├── train_model.py            # Full training pipeline: cleaning → tuning → calibration → SHAP artifacts → plots
├── data/
│   ├── heart.csv             # Raw combined dataset (1190 rows, as sourced)
│   └── heart_clean.csv       # Cleaned dataset used for training (918 rows)
├── models/                   # Generated by train_model.py
│   ├── heart_disease_model.pkl     # Calibrated model used for the displayed probability
│   ├── explainer_model.pkl         # Raw tree model used for SHAP explanations
│   ├── scaler.pkl
│   ├── feature_columns.pkl
│   ├── X_train_scaled_sample.pkl   # SHAP background data
│   ├── metadata.json               # Leaderboard + metrics, read by the app
│   └── leaderboard.csv
├── images/                   # Generated plots (EDA, ROC, confusion matrix, calibration, etc.)
├── requirements.txt
├── runtime.txt                # Pins Python 3.11 for cloud deployment
└── .streamlit/config.toml
```

## Running locally

```bash
pip install -r requirements.txt
python train_model.py        # regenerates models/ and images/ from data/heart.csv
streamlit run app.py
```
